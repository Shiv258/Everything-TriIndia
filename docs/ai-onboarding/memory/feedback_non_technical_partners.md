**Rule:** The 3 partners (Hansraj, Habib, Aftab) are not tech-savvy. In demos, dashboards, SOPs, training videos, and any partner-facing text: no jargon. No "API", "webhook", "OAuth", "cron" — say "connection", "auto-trigger", "permission", "scheduled task". Use analogies. Use stories like Priya's journey, not feature lists.

**Why:** They've never used a CRM, they've been burned by an Instagram agency that produced vanity metrics (10k followers, 0 bookings) and they've grown cynical of tech promises. Every deliverable must produce a tangible, demonstrable outcome. Owner-facing UIs need to be understandable by a 60-year-old who has never used a smartphone app in under 10 seconds.

**How to apply:** Anything partner-facing — Owner Dashboard copy, SOPs (Hindi + English), training videos, demo scripts — gets the no-jargon test. Internal code, commits, and Shiv-facing technical conversations are unaffected. Also: the Kalakar tradition is sacred — digitize the data layer only, don't propose changes to the Diwali reward ceremony or cash payouts.
